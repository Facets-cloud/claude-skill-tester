## Step 1: Preflight

The skill calls `preflight.sh aws-network-exposure`. Since I can't run that script directly, here's what it checks and what I'd run:

```
Checks:
  ✓ aws cli installed and on PATH
  ✓ active credentials (aws sts get-caller-identity)
  ✓ account ID, alias, default region
  ✓ CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 set (for TeamCreate)
  ✓ available AWS profiles (cat ~/.aws/config | grep '\[profile')
  ✓ EKS cluster discovery across all regions
```

**EKS discovery I'd run:**
```bash
# First: which regions are plausibly active?
aws ec2 describe-regions --query 'Regions[*].RegionName' --output text

# Then for each region:
aws eks list-clusters --region <region> --output json
```

If clusters found → `aws eks update-kubeconfig --name <cluster> --region <region>`

## Step 2: Survey

Before spawning any investigators, I map the landscape across all active regions. Commands I'd run per region:

```bash
# Security Groups with open inbound rules
aws ec2 describe-security-groups --region <r> \
  --query 'length(SecurityGroups)' --output text

# Load balancers
aws elbv2 describe-load-balancers --region <r> \
  --query 'length(LoadBalancers)' --output text

# Elastic IPs
aws ec2 describe-addresses --region <r> \
  --query 'length(Addresses)' --output text

# RDS instances
aws rds describe-db-instances --region <r> \
  --query 'length(DBInstances)' --output text

# S3 (global, once)
aws s3api list-buckets --query 'length(Buckets)'
```
