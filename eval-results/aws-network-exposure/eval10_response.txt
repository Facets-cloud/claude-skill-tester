## Step 4: Spawn Team

```
TeamCreate({ team_name: "aws-network-exposure" })
```

Three agents:
- **exposure-da** — devil's advocate / penetration tester mindset
- **investigator-infra-exposure** — checks SGs, EIPs, LBs, RDS `PubliclyAccessible` flag
- **investigator-data-exposure** — checks S3, KMS, SQS, ECR
- **investigator-k8s-exposure** — only if EKS kubeconfig was configured

For your specific RDS question, `investigator-infra-exposure` would run:

```bash
# Check PubliclyAccessible flag
aws rds describe-db-instances \
  --query 'DBInstances[*].{ID:DBInstanceIdentifier,Public:PubliclyAccessible,SGs:VpcSecurityGroups[*].VpcSecurityGroupId}' \
  --output table

# For each "PubliclyAccessible: true" instance, check its SG
aws ec2 describe-security-groups \
  --group-ids sg-xxxxxxxx \
  --query 'SecurityGroups[*].IpPermissions[?IpRanges[?CidrIp==`0.0.0.0/0`]]'
```

**Critical distinction the DA would enforce:**

| Situation | Verdict |
|-----------|---------|
| `PubliclyAccessible=true` + SG allows `0.0.0.0/0` on port 3306 | **CRITICAL** — directly reachable |
| `PubliclyAccessible=true` + SG only allows VPC CIDR | **MISCONFIGURED** but not exploitable — still a finding |
| `PubliclyAccessible=false` | Not internet-reachable regardless of SG |

The DA would reject any finding that doesn't show *both* the flag AND the SG evidence — because you need both vectors to be open for actual exposure.
